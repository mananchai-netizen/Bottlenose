# GitHub CLI Auth

Han uses GitHub CLI (`gh`) to create pull requests after it pushes a generated branch.

If `han start` can push code but fails at `gh pr create`, log in to GitHub CLI first:

```powershell
gh auth login
```

Recommended choices:

```text
GitHub.com
HTTPS
Login with a web browser
```

Verify the login:

```powershell
gh auth status
```

After `gh auth status` succeeds, `han start` can create pull requests automatically when a task generates file changes and pushes a branch.

If a branch was already pushed but the PR was not created, create the PR manually from the generated workspace or from GitHub.

Example:

```powershell
cd C:\Users\User\AppData\Local\Temp\han-workspaces\<task-page-id>
gh pr create --title "Task title" --body "Generated by Han AI" --head han/<task-page-id>
```

The `<task-page-id>` is the Notion task page id used by Han for the workspace and branch name.
